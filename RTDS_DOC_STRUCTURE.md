# RTDS 文档结构与分级目录

## 1. 一级目录（面向全员）

### 1.1 架构总览

- 系统目标与边界
- 运行形态（桌面端、仿真引擎、工具链）
- 关键目录地图（如 CMODEL_SOURCE、UDC_SOURCE、61850、WebSvcSupport）

### 1.2 构建与启动

- 构建入口（msbuild 任务）
- 启动入口（批处理脚本、Java 运行参数）
- 环境依赖（JDK、GCC、交叉编译器）

### 1.3 核心执行链路

- 启动到主流程
- 配置加载与初始化
- 仿真步进与周期计算
- 错误处理与日志路径

### 1.4 模型与算法

- C 模型生命周期（RAM / CODE）
- 控制模型、系统模型、共享代码
- 典型算法模板（排序、滤波、控制环）

### 1.5 协议与接口

- IEC 61850 相关目录与数据流
- 对外接口（Java/C DLL/脚本）

### 1.6 维护与排障

- 常见故障速查
- 调试抓手（日志、关键变量、断点建议）
- 变更影响评估模板

## 2. 二级目录（面向开发）

每个一级目录下建议固定结构：

1. 功能概述  
2. 关键文件列表  
3. 调用关系  
4. 示例代码与讲解  
5. 常见修改点与风险  
6. 测试与验证方法

## 3. 三级目录（面向维护负责人）

每个核心模块建议补充：

1. 设计约束与历史决策（Why）  
2. 性能与实时性指标（What）  
3. 演进路线与重构建议（How）

## 4. 章节模板（可复制）

```
# <模块名>

## 1. 模块职责
## 2. 关键文件
## 3. 执行流程
## 4. 代码示例
## 5. 常见问题
## 6. 变更注意事项
```
