# RTDS 代码说明总览（内部）

> 文档用途：帮助新员工快速理解 RTDS 代码结构、模块职责、关键执行路径与常见开发规范。  
> 保密级别：仅公司内部 Git 仓库可见，禁止外发。

## 1. 文档目标

1. 建立统一的代码认知框架，降低新人上手成本。  
2. 用“分层目录 + 关键路径 + 示例代码”解释系统行为。  
3. 将隐性经验转成显性文档，减少口口相传。

## 2. 读者分层

- L1（入职 1-2 周）：能定位代码、跑通基础流程、理解模块边界。  
- L2（入职 1-2 月）：能修改模块并评估影响范围。  
- L3（维护负责人）：能进行架构级优化与跨模块排障。

## 3. 推荐阅读路径（新人）

1. 先读文档地图：`RTDS_DOC_STRUCTURE.md`。  
2. 再读写作规范：`RTDS_DOC_STYLE_GUIDE.md`。  
3. 再看示例章节：`examples/CMODEL_SOURCE__m_sort.md` 与 `examples/STARTUP__runFromDos.md`。  
4. 最后进入各业务模块文档。

## 4. 建议文档目录

建议基于本仓库新增 `docs/`，并保持以下结构：

```
docs/
  RTDS_DOC_MASTER.md
  RTDS_DOC_STRUCTURE.md
  RTDS_DOC_STYLE_GUIDE.md
  01-Architecture/
  02-Build-And-Run/
  03-Core-Engine/
  04-Models/
  05-Protocols/
  06-Tools/
  07-Troubleshooting/
  examples/
```

## 5. 维护机制（建议）

1. 任何新增模块或重要逻辑变更，必须同步更新对应 md。  
2. 代码评审时增加文档检查项：是否补齐“示例 + 来源文件 + 风险说明”。  
3. 每月安排一次文档巡检，清理过期内容。

## 6. 内部发布与权限建议

1. 仓库设为公司内网 Git 私有仓。  
2. 按部门或项目组控制读写权限。  
3. 在 README 首行声明“内部资料，禁止外传”。  
4. 对含敏感参数或算法细节的章节采用更细粒度访问控制（若平台支持）。
