# RTDS 文档编写规范（内部）

## 1. 基础规则

1. 所有代码示例必须来自真实仓库文件，禁止伪造。  
2. 每个代码块后必须附“来源文件”。  
3. 每段说明必须回答三个问题：做什么、为什么、改哪里。  
4. 对新人不友好的术语，首次出现要给一句解释。

## 2. 代码示例标准格式

按以下顺序书写：

1. 场景说明（1-2 句）  
2. 代码块  
3. 来源文件（必填）  
4. 关键变量/函数解释  
5. 修改风险提示

示例模板：

```markdown
### 示例：<标题>

场景：<这个片段在什么流程里被执行>

```c
<代码片段>
```

来源文件：<目录/文件名>

说明：
- <关键点 1>
- <关键点 2>

风险：
- <修改后可能影响>
```

## 3. 文件引用规范

建议统一写法：

- 来源文件：CMODEL_SOURCE/_m_sort.c  
- 相关头文件：CMODEL_SOURCE/_m_sort.h  
- 启动脚本：runFromDos.bat

若引用多个文件，按“主文件在前、依赖文件在后”排列。

## 4. 变更记录规范

每个文档末尾维护变更记录：

```markdown
## 变更记录

- 2026-06-03：初版建立，补充启动流程与排序模型示例。
```

## 5. 评审清单（PR 可直接复用）

1. 是否有“模块职责”说明。  
2. 是否有“代码示例 + 来源文件”。  
3. 是否标注了风险与验证方法。  
4. 是否包含最新变更记录。
